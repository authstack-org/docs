---
title: Admin Panel
description: Manage applications, operators, and tenant users through the instance admin UI.
---

The admin panel at `/admin/login` is for **Authstack instance operators** only. Tenant end users never use this UI.

## Operator roles

| Role | Access |
| --- | --- |
| **Instance admin** | Full instance access: all applications, operator management, application deletion |
| **App admin** | Scoped to assigned applications: view app details, list users, create tenant users |

The first instance admin is created with [`authstack bootstrap-admin`](/docs/getting-started#bootstrap-the-first-instance-admin).

Additional operators are created by an instance admin from **Operators → Add operator**.

## Applications

From the dashboard, instance admins can create applications and view one-time `client_secret` credentials.

All operators with access to an application can open its detail page to:

- View app metadata and user count
- Open **Manage users** to list tenant users
- **Add user** to provision a tenant account (name, email, password)

Instance admins can also delete an application from the app detail page. This removes all tenant data for that application.

## What the admin panel is not

- Not a login portal for tenant end users
- Not a replacement for your product's team-management UI
- Not org/role management for tenants (use the [Members API](/docs/api/members) from your BFF for that)

## JSON API

Instance admins can still create applications programmatically after logging in:

```bash
curl -c cookies.txt -X POST http://localhost:8080/admin/login \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "email=admin@example.com&password=your-password"

curl -b cookies.txt -X POST http://localhost:8080/admin/applications \
  -H "Content-Type: application/json" \
  -d '{"name": "my-app"}'
```

App admins receive `403 Forbidden` from this JSON endpoint.
