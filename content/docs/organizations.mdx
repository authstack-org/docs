---
title: Organizations
description: App-scoped organizations for tenants, teams, and collaboration in Authstack.
---

Authstack **organizations** are **scoped to a single application**. They are how your app models tenants, teams, or shared workspaces. Authstack does not create an organization automatically when a user signs up.

For per-user private data, use the user's id (`sub` in the JWT) directly in your application — you do not need a personal organization wrapper.

## Directory vs organization

| Layer | Scope | Purpose |
| --- | --- | --- |
| **Directory** | Authstack identity realm | One email = one user across apps in that directory |
| **Organization** | Single application | Tenant boundary, team, or workspace **inside that app** |

The same person (`alice@acme.com`) in one directory can belong to different organizations in App A and App B. Those org graphs are independent.

## Creating organizations

Create an organization via the API (`POST /orgs`) or from the admin panel (**Organizations → New organization**). Slugs are unique **per application**.

Users must already have a `user_app_grant` for the application before they can be added as members (signup, invite accept, or admin provisioning).

## Members and roles

- Add members with `POST /orgs/{id}/members` or from the admin organization detail page.
- Member `role` is a string defined by your app (`owner`, `admin`, `member`, etc.).
- Organization invites add users to a specific org; see [Organization invites](/docs/invites).

## JWT and org context

Access tokens include `org_id` and `role` **only when** the user has an active organization membership for the issuing application. After signup or login with no org memberships, the token carries user identity (`sub`, `email`, `app_id`, `directory_id`) without org claims.

Use `POST /auth/switch-org` to issue a new token pair for a different organization the user belongs to. Use `POST /auth/refresh` with an optional `org_id` to rotate tokens while selecting org context.

## Typical patterns

| Your product | How to use organizations |
| --- | --- |
| B2B SaaS (one app) | One org per customer; all users are members |
| Multi-app suite | Same directory user; separate org memberships per app |
| Solo / B2C | Skip orgs; key resources by `user_id` |
