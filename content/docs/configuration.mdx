---
title: Configuration
description: Environment variables and runtime settings for Authstack.
---

Authstack is configured with environment variables.

## API server

| Variable | Required | Description |
| --- | --- | --- |
| `DATABASE_URL` | Yes | PostgreSQL connection string. |
| `JWT_PRIVATE_KEY` | Yes | Base64-encoded ES256 private key generated by `make keys`. |
| `JWT_PUBLIC_KEY` | Yes | Base64-encoded ES256 public key generated by `make keys`. |
| `ACCESS_TOKEN_EXPIRY_SECS` | No | Access token lifetime in seconds. Defaults to `900`. |
| `REFRESH_TOKEN_EXPIRY_SECS` | No | Refresh token lifetime in seconds. Defaults to `2592000`. |
| `PORT` | No | HTTP port. Defaults to `8080`. |
| `RUST_LOG` | No | Rust log filter, for example `authstack=debug` or `authstack=info`. |

## Bootstrap (first instance admin)

These variables are only used by `authstack bootstrap-admin` or the Docker entrypoint on a **fresh database**. They are not required to run the API after the first admin exists.

| Variable | Required | Description |
| --- | --- | --- |
| `AUTHSTACK_BOOTSTRAP_EMAIL` | Bootstrap | Email for the first instance admin. |
| `AUTHSTACK_BOOTSTRAP_PASSWORD` | Bootstrap | Password for the first instance admin. |

Create the first admin manually:

```bash
AUTHSTACK_BOOTSTRAP_EMAIL=admin@example.com \
AUTHSTACK_BOOTSTRAP_PASSWORD='your-strong-password' \
  authstack bootstrap-admin
```

Or pipe the password:

```bash
echo 'your-strong-password' | authstack bootstrap-admin \
  --email admin@example.com \
  --password-stdin
```

The command refuses to run when any `admin_user` row already exists.

## JWT keys

Run:

```bash
make keys
```

Paste the generated `JWT_PRIVATE_KEY` and `JWT_PUBLIC_KEY` values directly into your environment. The values are base64-encoded PEM files and should remain single-line strings.

## Logging

For production, start with:

```txt
RUST_LOG=authstack=info
```

For debugging local issues, use:

```txt
RUST_LOG=authstack=debug
```
