title: Authentication

## Application authentication

All `/auth/*`, `/users`, `/orgs`, and `/orgs/:id/members` endpoints require HTTP Basic auth from the application backend.

```txt
Authorization: Basic base64(<app_id>:<client_secret>)
```

The `app_id` and `client_secret` are returned when an admin creates an application. The `client_secret` is only returned once, so store it immediately in the consuming backend's secret store.

## Admin authentication

Authstack ships with a browser-based admin panel at `/admin/login`.

The first admin account is bootstrapped with `AUTHSTACK_ADMIN_KEY`:

```bash
curl -X POST http://localhost:8080/admin/users \
  -H "Content-Type: application/json" \
  -H "X-Admin-Key: <AUTHSTACK_ADMIN_KEY>" \
  -d '{"email": "admin@example.com", "password": "your-password"}'
```

After the first admin exists, log in through:

```txt
http://localhost:8080/admin/login
```

The admin panel stores the session token in an `HttpOnly; SameSite=Strict` cookie.

## User tokens

`POST /auth/login` returns an access token, refresh token, and token type.

```json
{
  "access_token": "<jwt>",
  "refresh_token": "<jwt>",
  "token_type": "Bearer"
}
```

The access token payload includes the user, app, active organization, organization type, role, email, token ID, issued-at time, and expiration.

```json
{
  "sub": "usr_01j4hz0r3fexwpbgm41z1w57at",
  "app_id": "app_01j4hz0r3fexwpbgm41z1w57at",
  "org_id": "org_01j4hz0r3fexwpbgm41z1w57at",
  "org_type": "personal",
  "role": "owner",
  "email": "alice@example.com",
  "jti": "<uuid>",
  "iat": 1234567890,
  "exp": 1234568790
}
```

## Refresh token rotation

Every refresh request invalidates the old refresh token and returns a new token pair. Reusing a rotated refresh token returns `401`.

`POST /auth/refresh` can optionally include `org_id` to issue the new access token for a specific organization the user belongs to.

## JWKS

Consuming services can fetch Authstack' public key set:

```txt
GET /.well-known/jwks.json
```

Use the JWKS endpoint to verify Authstack-issued JWTs locally without calling Authstack on every request.
